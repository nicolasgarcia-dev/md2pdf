    Custom CSS — How it works
    
                
            
    

            
    
                
    
                    Your CSS is loaded after the default styles, so you can override
                    anything. It applies to both the live preview and the downloaded PDF.
                
    

                
    
                    
    Available fonts (pre-installed on the server)
    
                    
    
                        Noto Sans
                        Noto Serif
                        Inter
                        EB Garamond
                        Roboto Slab
                        JetBrains Mono
                    
    
                
    

                
    
                    
    Key selectors
    
                    
    
                        
                            
    
                                Selector
                                What it controls
                            
    
                            
                        
                            
    
                                html, body
                                Font family, text colour, page background (use with @page { background-color } for full coverage)
                            
    
                            
    
                                h1 – h4
                                Heading size, weight, colour, border
                            
    
                            
    
                                a
                                Link colour and decoration
                            
    
                            
    
                                p
                                Paragraph spacing and alignment
                            
    
                            
    
                                blockquote
                                Quote block border, background, italic
                            
    
                            
    
                                pre, .codehilite
                                Code block background and font
                            
    
                            
    
                                code
                                Inline code chip colour
                            
    
                            
    
                                table, th, td
                                Table borders and cell padding
                            
    
                            
    
                                nav.toc
                                Table of contents box
                            
    
                            
    
                                main.document
                                Main content wrapper
                            
    
                            
                    
    
                
    

                
    
                    
    Useful PDF-specific properties
    
                    
    
                        
                            
    
                                Property
                                Effect
                            
    
                            
                        
                            
    
                                body { padding: … }
                                Page margins (e.g. 15mm 20mm) — default is 20mm 18mm
                            
    
                            
    
                                page-break-inside: avoid
                                Keeps an element on one page
                            
    
                            
    
                                page-break-after: avoid
                                Prevents a page break after the element
                            
    
                            
    
                                text-align: justify
                                Justified paragraph text
                            
    
                            
                    
    
                
    

                
    
                    
    Example — full page background colour
    
                    
    
                        One rule is enough — the server automatically extends the colour to the margin area so the entire page (including gutters) gets the same fill. No need to write an @page rule yourself.
                    
    
                    
    /* Warm cream paper — keep code, tables and quotes
   on the same warm palette so nothing falls back
   to the default light-blue defaults. */
html, body { background: #fdf6e3; color: #2a2a2a; }
a { color: #b58900; }
blockquote { border-left-color: #d8c98c; background: #faecc6; color: #5a4d22; }
code { background: #faecc6; color: #5a4d22; border: 1px solid #ead49a; }
pre, .codehilite {
    background: #faecc6;
    color: #2a2a2a;
    border: 1px solid #ead49a;
}
th { background: #faecc6; color: #2a2a2a; }

/* Slate dark page — recolor every neighbouring
   surface so the page reads as one block. */
html, body { background: #1e1e2e; color: #cdd6f4; }
h1, h2, h3 { color: #89b4fa; }
a { color: #89dceb; }
blockquote { border-left-color: #585b70; background: #2a2b3c; color: #cdd6f4; }
code { background: #2a2b3c; color: #f5c2e7; border: 1px solid #45475a; }
pre, .codehilite {
    background: #181825;
    color: #cdd6f4;
    border: 1px solid #313244;
}
th { background: #2a2b3c; color: #cdd6f4; border-color: #45475a; }
td { border-color: #313244; }
hr { border-top: 1px solid #313244; }
    
                
    

                
    
                    
    Example — dark academia style
    
                    
    body {
    font-family: "Noto Serif", Georgia, serif;
    color: #1c1410;
    background: #fdf8f0;
}

h1, h2, h3 {
    font-family: "Inter", sans-serif;
    color: #7b2d00;
    letter-spacing: -0.02em;
}

h1 { border-bottom: 2px solid #c9834a; }

a { color: #7b2d00; text-decoration: underline; }

blockquote {
    border-left: 4px solid #c9834a;
    background: #fdf0e0;
    font-style: italic;
}

/* Match the warm palette in code, tables and the TOC. */
code {
    background: #fdf0e0;
    color: #5a3325;
    border: 1px solid #f0dabb;
}
pre, .codehilite {
    background: #fbf2e1;
    color: #2a201a;
    border: 1px solid #f0dabb;
}
th { background: #f4e3c4; color: #1c1410; border-color: #d8b58a; }
nav.toc { background: #fdf0e0; border: 1px solid #f0dabb; }
    
                
    

                
    
                    
    Task list checkboxes
    
                    
    
                        Use - [ ] and - [x] in your Markdown to render checkboxes. The single-line CSS below recolours them and works the same in the live preview and the downloaded PDF.
                    
    
                    
    /* One-liner — recolours the native check */
input[type="checkbox"] { accent-color: #16a34a; }

/* Slightly bigger and a touch of breathing room */
input[type="checkbox"] {
    accent-color: #2563eb;
    transform: scale(1.15);
    margin-right: 0.55em;
}
    
                
    

                
    
                    
    Tips
    
                    
      
                        
    • Changes apply to the preview in real time. The PDF is only generated on download.
      • 
                        
    • To colour the full page background, write html, body { background: #colour; } — the server automatically fills the margin area with the same colour so the entire page looks uniform.
      • 
                        
    • Your CSS is saved automatically in the browser between sessions.
      • 
                    
    
                
    

                
    
                    
    Styling code blocks
    
                    
    
                        Fenced code blocks render as <div class="codehilite"><pre>…</pre></div> when a language is recognised, or just <pre><code>…</code></pre> otherwise. Style both pre and .codehilite together to be safe.
                    
    
                    
    Recreate the default boxed look
    
                    
    pre, .codehilite {
    background: #f6f8fa;
    border-radius: 6px;
}
pre {
    padding: 0.8em 1em;
}
    
                    
    Boxed code with a border
    
                    
    pre, .codehilite {
    background: #fafafa;
    border: 1px solid #e2e2e2;
    border-radius: 6px;
}
pre { padding: 0.8em 1em; }
    
                    
    Dark code block (with readable tokens)
    
                    
    pre, .codehilite {
    background: #1e1e2e;
    color: #cdd6f4;
    border: 1px solid #313244;
    border-radius: 6px;
    /* Override the syntax palette so red/blue tokens
       don't disappear against the dark background. */
    --syn-keyword: #cba6f7;
    --syn-string:  #a6e3a1;
    --syn-comment: #7f849c;
    --syn-number:  #fab387;
    --syn-function:#89b4fa;
    --syn-builtin: #f9e2af;
    --syn-operator:#94e2d5;
}
pre { padding: 0.9em 1.1em; }
    
                    
    Inline `code` chips
    
                    
    code {
    background: #eef2ff;
    color: #3730a3;
    padding: 0.1em 0.35em;
    border-radius: 3px;
    font-size: 0.92em;
}
    
                
    

                
    
                    
    Syntax highlighting
    
                    
    
                        The fastest way to recolor highlighted code is to set the palette variables on pre, .codehilite. They drive both the live preview (highlight.js) and the downloaded PDF (Pygments) at once, so you don't need to write two sets of rules.
                    
    
                    
    Recommended — set the palette variables
    
                    
    pre, .codehilite {
    --syn-keyword:  #cf222e;
    --syn-string:   #0a3069;
    --syn-comment:  #6e7781;
    --syn-number:   #0550ae;
    --syn-function: #8250df;
    --syn-builtin:  #0550ae;
    --syn-operator: #cf222e;
    --syn-type:     #0550ae;
    --syn-attr:     #0550ae;
    --syn-meta:     #8250df;
}
    
                    
    
                        If you need finer control you can still target individual Pygments classes directly. Each token is wrapped in a <span> with a short class — always scope rules with .codehilite so they don't leak elsewhere.
                    
    
                    
    
                        
                            
    
                                Class
                                Token
                            
    
                            
                        
                            
    .k .kd .kn .krKeywords (def, class, import…)
    
                            
    .ktBuilt-in types (int, string)
    
                            
    .s .s1 .s2Strings ('…', "…")
    
                            
    .seString escapes (\n)
    
                            
    .c .c1 .cmComments (line / block)
    
                            
    .cpPreprocessor directives
    
                            
    .nfFunction names
    
                            
    .ncClass names
    
                            
    .nbBuilt-ins (print, len)
    
                            
    .nvVariables
    
                            
    .ndDecorators (@foo)
    
                            
    .ntHTML / XML tags
    
                            
    .naAttributes
    
                            
    .o .owOperators (+, and)
    
                            
    .m .mi .mfNumbers (int / float)
    
                            
    .pPunctuation
    
                            
    .gi .gdDiff added / removed lines
    
                            
                    
    
                    
    Example — Catppuccin Mocha-flavored block
    
                    
    pre, .codehilite {
    background: #1e1e2e;
    color: #cdd6f4;
    border: 1px solid #313244;
    border-radius: 6px;
    padding: 0.9em 1.1em;
    --syn-keyword:  #cba6f7;
    --syn-string:   #a6e3a1;
    --syn-comment:  #7f849c;
    --syn-number:   #fab387;
    --syn-function: #89b4fa;
    --syn-builtin:  #f9e2af;
    --syn-operator: #94e2d5;
    --syn-type:     #f9e2af;
    --syn-attr:     #fab387;
    --syn-meta:     #cba6f7;
}
    
                    
    
                        Need to nudge only one token? Layer a single rule on top, e.g.
                        .codehilite .c1 { color: #888; font-style: italic; }.

    Page layout & print control
    
                    
    
                        
                            
    
                                Rule
                                Effect
                            
    
                            
                        
                            
    @page { size: A4; margin: 18mm 16mm; }Override page size and margins (also Letter, Legal, landscape)
    
                            
    @page { background-color: #colour; }Fill the entire page including margin gutters
    
                            
    @page :first { … }Style only the first page (e.g. cover)
    
                            
    page-break-before: alwaysForce a new page before the element
    
                            
    page-break-after: avoidKeep the next block on the same page (great for headings)
    
                            
    page-break-inside: avoidKeep the element together on one page
    
                            
    orphans: 3; widows: 3;Minimum lines kept together at page breaks
    
                            
                    
    
                
    

                
    
                    
    Auto-numbered headings
    
                    
    
                        CSS counters can prefix every heading with a numeric label automatically — this is how the Academic theme produces output like 1., 1.1, 1.2. You can drop the snippet below into Custom CSS to get the same effect on any other theme. Works identically in the preview and the PDF.
                    
    
                    
    /* Reset the counter on the document root,
   then re-reset h3 every time h2 changes. */
main.document               { counter-reset: h2counter; }
main.document h2            { counter-reset: h3counter; }

main.document h2::before {
    counter-increment: h2counter;
    content: counter(h2counter) ". ";
}

main.document h3::before {
    counter-increment: h3counter;
    content: counter(h2counter) "." counter(h3counter) " ";
}
    
                    
    
                        Add an h4 level the same way, or drop the h3 rule if you only want top-level numbering. ::before is what places the number; the counter-reset on h2 ensures every section restarts its sub-counter.
                    
    
                
    

                
    
                    
    Page numbers
    
                    
    
                        CSS Paged Media defines @page margin boxes (@top-left, @bottom-center, …) with the counter(page) and counter(pages) counters. These rules only render in the PDF (not the live preview, which has no notion of pages):
                    
    
                    
    @page {
    @bottom-center {
        content: "Page " counter(page) " of " counter(pages);
        font-family: "Inter", sans-serif;
        font-size: 9pt;
        color: #888;
    }
}

/* No numbers on the cover page (optional) */
@page :first {
    @bottom-center { content: ""; }
}

/* Different positions are also valid */
@page {
    @top-right { content: counter(page); font-size: 9pt; color: #888; }
}
    
                    
    
                        Engine support: the WeasyPrint fallback honours these rules in full. The primary Chromium renderer has only partial support — depending on the document, page numbers via @page margin boxes may or may not appear. The simplest way to test it: paste the snippet, download the PDF, and check the corner of every page.
                    
    
                
    

                
    
                    
    Specificity & cascade tips
    
                    
      
                        
    • Your CSS loads after the built-in styles and the syntax-highlighting stylesheet, so plain selectors like pre { … } usually win.
      • 
                        
    • If a default rule still wins, it's because of higher specificity. Match it: replace pre with pre, .codehilite or use .codehilite pre.
      • 
                        
    • As a last resort, append !important to a single declaration — but try a more specific selector first.
      • 
                        
    • Use any web-safe CSS unit: px, em, rem, %, plus print units mm, cm, pt, in.
      • 
                        
    • CSS custom properties work too: define them on html and reuse with var(--name).
      •
