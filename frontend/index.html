    Custom CSS — How it works
    
                
            
    

            
    
                
    
                    Your CSS is loaded after the default styles, so you can override
                    anything. It applies to both the live preview and the downloaded PDF.
                
    

                
    
                    
    Available fonts (pre-installed on the server)
    
                    
    
                        Noto Sans
                        Noto Serif
                        Inter
                        JetBrains Mono
                    
    
                
    

                
    
                    
    Key selectors
    
                    
    
                        
                            
    
                                Selector
                                What it controls
                            
    
                            
                        
                            
    
                                html, body
                                Font family, text colour, page background (use with @page { background-color } for full coverage)
                            
    
                            
    
                                h1 – h4
                                Heading size, weight, colour, border
                            
    
                            
    
                                a
                                Link colour and decoration
                            
    
                            
    
                                p
                                Paragraph spacing and alignment
                            
    
                            
    
                                blockquote
                                Quote block border, background, italic
                            
    
                            
    
                                pre, .codehilite
                                Code block background and font
                            
    
                            
    
                                code
                                Inline code chip colour
                            
    
                            
    
                                table, th, td
                                Table borders and cell padding
                            
    
                            
    
                                nav.toc
                                Table of contents box
                            
    
                            
    
                                main.document
                                Main content wrapper
                            
    
                            
                    
    
                
    

                
    
                    
    Useful PDF-specific properties
    
                    
    
                        
                            
    
                                Property
                                Effect
                            
    
                            
                        
                            
    
                                body { padding: … }
                                Page margins (e.g. 15mm 20mm) — default is 20mm 18mm
                            
    
                            
    
                                page-break-inside: avoid
                                Keeps an element on one page
                            
    
                            
    
                                page-break-after: avoid
                                Prevents a page break after the element
                            
    
                            
    
                                text-align: justify
                                Justified paragraph text
                            
    
                            
                    
    
                
    

                
    
                    
    Example — full page background colour
    
                    
    
                        One rule is enough — the server automatically extends the colour to the margin area so the entire page (including gutters) gets the same fill. No need to write an @page rule yourself.
                    
    
                    
    /* Warm cream paper */
html, body { background: #fdf6e3; }

/* Slate dark page */
html, body { background: #1e1e2e; color: #cdd6f4; }
h1, h2, h3 { color: #89b4fa; }
a { color: #89dceb; }
    
                
    

                
    
                    
    Example — dark academia style
    
                    
    body {
    font-family: "Noto Serif", Georgia, serif;
    color: #1c1410;
    background: #fdf8f0;
}

h1, h2, h3 {
    font-family: "Inter", sans-serif;
    color: #7b2d00;
    letter-spacing: -0.02em;
}

h1 { border-bottom: 2px solid #c9834a; }

a { color: #7b2d00; text-decoration: underline; }

blockquote {
    border-left: 4px solid #c9834a;
    background: #fdf0e0;
    font-style: italic;
}
    
                
    

                
    
                    
    Tips
    
                    
      
                        
    • Changes apply to the preview in real time. The PDF is only generated on download.
      • 
                        
    • To colour the full page background, write html, body { background: #colour; } — the server automatically fills the margin area with the same colour so the entire page looks uniform.
      • 
                        
    • Your CSS is saved automatically in the browser between sessions.
      • 
                    
    
                
    

                
    
                    
    Styling code blocks
    
                    
    
                        Fenced code blocks render as <div class="codehilite"><pre>…</pre></div> when a language is recognised, or just <pre><code>…</code></pre> otherwise. Style both pre and .codehilite together to be safe.
                    
    
                    
    Recreate the default boxed look
    
                    
    pre, .codehilite {
    background: #f6f8fa;
    border-radius: 6px;
}
pre {
    padding: 0.8em 1em;
}
    
                    
    Boxed code with a border
    
                    
    pre, .codehilite {
    background: #fafafa;
    border: 1px solid #e2e2e2;
    border-radius: 6px;
}
pre { padding: 0.8em 1em; }
    
                    
    Dark code block
    
                    
    pre, .codehilite {
    background: #1e1e2e;
    color: #cdd6f4;
    border-radius: 6px;
}
pre { padding: 0.9em 1.1em; }
    
                    
    Inline `code` chips
    
                    
    code {
    background: #eef2ff;
    color: #3730a3;
    padding: 0.1em 0.35em;
    border-radius: 3px;
    font-size: 0.92em;
}
    
                
    

                
    
                    
    Syntax highlighting
    
                    
    
                        Recognised languages are tokenised by Pygments. Each token is wrapped in a <span> with a short class. Always scope rules with .codehilite so they only affect highlighted blocks.
                    
    
                    
    
                        
                            
    
                                Class
                                Token
                            
    
                            
                        
                            
    .k .kd .kn .krKeywords (def, class, import…)
    
                            
    .ktBuilt-in types (int, string)
    
                            
    .s .s1 .s2Strings ('…', "…")
    
                            
    .seString escapes (\n)
    
                            
    .c .c1 .cmComments (line / block)
    
                            
    .cpPreprocessor directives
    
                            
    .nfFunction names
    
                            
    .ncClass names
    
                            
    .nbBuilt-ins (print, len)
    
                            
    .nvVariables
    
                            
    .ndDecorators (@foo)
    
                            
    .ntHTML / XML tags
    
                            
    .naAttributes
    
                            
    .o .owOperators (+, and)
    
                            
    .m .mi .mfNumbers (int / float)
    
                            
    .pPunctuation
    
                            
    .gi .gdDiff added / removed lines
    
                            
                    
    
                    
    Example — full custom colour scheme
    
                    
    pre, .codehilite {
    background: #1e1e2e;
    color: #cdd6f4;
    border-radius: 6px;
}
pre { padding: 0.9em 1.1em; }

.codehilite .k,
.codehilite .kd,
.codehilite .kn  { color: #cba6f7; font-weight: bold; } /* keywords */
.codehilite .s,
.codehilite .s1,
.codehilite .s2  { color: #a6e3a1; }                    /* strings */
.codehilite .c,
.codehilite .c1,
.codehilite .cm  { color: #6c7086; font-style: italic; }/* comments */
.codehilite .nf  { color: #89b4fa; }                    /* functions */
.codehilite .nc  { color: #f9e2af; }                    /* classes */
.codehilite .nb  { color: #fab387; }                    /* builtins */
.codehilite .m,
.codehilite .mi,
.codehilite .mf  { color: #fab387; }                    /* numbers */
.codehilite .o   { color: #94e2d5; }                    /* operators */
    
                    
    
                        To tweak just one aspect (e.g. only comments), override that single class:
                        .codehilite .c1 { color: #888; font-style: italic; }

    Page layout & print control
    
                    
    
                        
                            
    
                                Rule
                                Effect
                            
    
                            
                        
                            
    @page { size: A4; margin: 18mm 16mm; }Override page size and margins (also Letter, Legal, landscape)
    
                            
    @page { background-color: #colour; }Fill the entire page including margin gutters
    
                            
    @page :first { … }Style only the first page (e.g. cover)
    
                            
    page-break-before: alwaysForce a new page before the element
    
                            
    page-break-after: avoidKeep the next block on the same page (great for headings)
    
                            
    page-break-inside: avoidKeep the element together on one page
    
                            
    orphans: 3; widows: 3;Minimum lines kept together at page breaks
    
                            
                    
    
                
    

                
    
                    
    Specificity & cascade tips
    
                    
      
                        
    • Your CSS loads after the built-in styles and the syntax-highlighting stylesheet, so plain selectors like pre { … } usually win.
      • 
                        
    • If a default rule still wins, it's because of higher specificity. Match it: replace pre with pre, .codehilite or use .codehilite pre.
      • 
                        
    • As a last resort, append !important to a single declaration — but try a more specific selector first.
      • 
                        
    • Use any web-safe CSS unit: px, em, rem, %, plus print units mm, cm, pt, in.
      • 
                        
    • CSS custom properties work too: define them on html and reuse with var(--name).
      •
